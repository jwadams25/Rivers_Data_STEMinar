What is EDA?

Exploratory Data Analysis is the process of using visualizations and summary statistics to better understand the structure, patterns, and relationships in your data.

The goal is to ask questions and let the data guide your understanding.

Key Questions to Ask

When you first encounter a dataset, always ask:

  • Where did this data come from?
  • What does each row represent? (observational unit)
  • How many observations are there?
  • How many variables are there?
  • What types of variables are present? (numerical vs. categorical)

For loan50: 50 observations, 18 variables from Lending Club.

Cleaning the Data

Step 1: Select Relevant Variables

We don’t need all 18 variables. Let’s narrow it down to 9:

loans <- loan50 |>
  select(loan_amount, interest_rate, term, grade,
         state, total_income, homeownership, loan_purpose, total_credit_limit)

What happened?

  • Created a new object called loans
  • Used the pipe (|>) to pass loan50 into select()
  • Kept only the 9 columns we care about

Step 2: Convert Factor Variables

Some variables are stored as fct (factor) — let’s convert to character:

loans <- loans |>
  mutate_if(is.factor, as.character)

Now we’re ready to explore!

Exploring Loan Amounts

Question: How big do loans tend to be?

To answer this, we’ll use:

  • A histogram to visualize the distribution
  • Summary statistics to quantify the center and spread

Building a Histogram

loans |>
  ggplot(aes(x = loan_amount)) +
  geom_histogram() +
  labs(
    x = "Loan Amount ($)",
    y = "Count",
    title = "Distribution of Loan Amounts"
  ) +
  theme_minimal()

Building a Histogram

Anatomy of a ggplot Call

data |>                          # Start with data
  ggplot(aes(x = variable)) +   # Set up the canvas
  geom_histogram() +             # Add the geometry
  labs(x = "", y = "",           # Add labels
       title = "") +
  theme_minimal()                # Apply a clean theme

Each piece is a layer added with +

Summary Statistics

loans |>
  filter(!is.na(loan_amount)) |>
  summarise(
    mean = mean(loan_amount),
    median = median(loan_amount),
    sd = sd(loan_amount),
    Q1 = quantile(loan_amount, 0.25),
    Q3 = quantile(loan_amount, 0.75)
  )
# A tibble: 1 × 5
   mean median     sd    Q1    Q3
  <dbl>  <dbl>  <dbl> <dbl> <dbl>
1 17083  15500 10455.  7125 24000

Reading the Distribution

When describing a distribution, always mention:

  • Shape: Is it symmetric, skewed left, or skewed right?
  • Center: What is the typical value? (mean and median)
  • Spread: How variable are the values? (standard deviation, IQR)
  • Outliers: Are there any unusual observations?

Comparing Groups

Question: Does loan size differ by homeownership?

We can split our histogram by a categorical variable using facet_wrap().

Faceted Histogram

loans |>
  ggplot(aes(x = loan_amount)) +
  geom_histogram() +
  labs(
    x = "Loan Amount ($)",
    y = "Count",
    title = "Loan Amounts by Homeownership Status"
  ) +
  theme_minimal() +
  facet_wrap(~homeownership, nrow = 3)

Faceted Histogram

Grouped Summary Statistics

Add group_by() before summarise() to get stats per group:

loans |>
  filter(!is.na(loan_amount)) |>
  group_by(homeownership) |>
  summarise(
    mean = mean(loan_amount),
    median = median(loan_amount),
    sd = sd(loan_amount),
    Q1 = quantile(loan_amount, 0.25),
    Q3 = quantile(loan_amount, 0.75)
  )

Use mutate() to calculate the proportion:

loans |>

  arrange(desc(n)) |>

Exploring Relationships

Question: Does income predict loan size?

We use a scatter plot when we have two numerical variables:

  • Predictor (x-axis): total_income
  • Outcome (y-axis): loan_amount

Scatter Plot

loans |>
  ggplot(aes(x = total_income, y = loan_amount)) +
  geom_point() +
  labs(
    x = "Total Income ($)",
    y = "Loan Amount ($)",
    title = "Total Income vs. Loan Amount"
  ) +
  theme_minimal()

Scatter Plot

Adding a Linear Model

Use geom_smooth(method = lm) to overlay a regression line:

loans |>
  ggplot(aes(x = total_income, y = loan_amount)) +
  geom_point() +
  geom_smooth(method = lm, se = FALSE) +
  labs(
    x = "Total Income ($)",
    y = "Loan Amount ($)",
    title = "Total Income vs. Loan Amount with Linear Model"
  ) +
  theme_minimal()

Adding a Linear Model

Getting the Equation

The lm() function gives us the intercept and slope:

lm(data = loans, formula = loan_amount ~ total_income)

Call:
lm(formula = loan_amount ~ total_income, data = loans)

Coefficients:
 (Intercept)  total_income  
   8.443e+03     8.211e-02

The general form of the equation is:

\[\hat{y} = b_0 + b_1 \times x\]

Correlation and R-Squared

loans |>
  summarise(
    mean_income = mean(total_income),
    sd_income = sd(total_income),
    mean_loan = mean(loan_amount),
    sd_loan = sd(loan_amount),
    r = cor(loan_amount, total_income)
  ) |>
  mutate(
    r_squared = r^2
  )
# A tibble: 1 × 6
  mean_income sd_income mean_loan sd_loan     r r_squared
        <dbl>     <dbl>     <dbl>   <dbl> <dbl>     <dbl>
1     105221.    68142.     17083  10455. 0.535     0.286

Interpreting the Model

Slope → Direction of the association

  • Positive slope = as x increases, y tends to increase
  • Negative slope = as x increases, y tends to decrease

R-squared → Strength of the association

  • Tells us the proportion of variability in y explained by x
  • Closer to 1 = stronger linear relationship
  • Closer to 0 = weaker linear relationship

Categorical Data

Question: Why are people getting loans?

For categorical data, we use bar charts and frequency tables.

Bar Chart of Loan Purpose

loans |>
  ggplot(aes(y = fct_rev(fct_infreq(loan_purpose)))) +
  geom_bar() +
  labs(
    x = "Count",
    y = "Loan Purpose",
    title = "Purposes of Loans in Our Sample"
  ) +
  theme_minimal()

Bar Chart of Loan Purpose

A Helpful Trick: fct_infreq()

aes(y = fct_rev(fct_infreq(loan_purpose)))
  • fct_infreq() → orders the categories by frequency
  • fct_rev() → reverses the order so the most frequent is at the top

This makes bar charts much easier to read!

Key Takeaways

The EDA Workflow

  1. Load your libraries and data
  2. Inspect the data (glimpse(), view())
  3. Clean the data (select(), mutate())
  4. Ask a question
  5. Visualize (histogram, bar chart, scatter plot)
  6. Summarize (mean, median, sd, correlation)
  7. Interpret and communicate your findings
  8. Repeat with new questions!
